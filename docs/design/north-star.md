## 各层定义

### 第一层：数据层（地基）

日频自动更新的原始数据，为上层分析提供输入。

| 数据类型 | 来源 | 存储 | 更新频率 |
|---------|------|------|---------|
| 量价 | FMP API | market.db:daily_price | 日频 (云端 cron) |
| 基本面 | FMP API | market.db:income/BS/CF/ratios/metrics | 周频 (云端 cron) |
| 期权 IV | MarketData.app | market.db:iv_daily | 日频 (云端 cron) |
| 宏观 | FRED API | macro_snapshot.json (4h/12h TTL) | 准实时 |
| 前瞻预期 | yfinance | market.db:forward_estimates/metadata | 周频 (云端 cron) |
| 社交情感 | Adanos (Reddit + X) | market.db:social_sentiment | 日频 (云端 cron 06:55) |
| 新闻 | FMP API | 按需获取 | 按需 |

### 第二层：分析层（三维度）

对原始数据的深加工，产出每只股票在三个维度的分析结论。

**基本面分析（季频）**
- 驱动方式：Knowledge 层（6 lens + 辩论协议 + memo 模板）
- 产出：深度研究报告 + 基本面评分
- 更新触发：财报发布、重大事件、分析过期
- 当前状态：成熟（15 agent deep pipeline）

**技术分析（日频）**
- 驱动方式：经因子研究验证过的技术指标
- 产出：技术面信号（趋势、动量、超买超卖、量能异常）
- 更新触发：每日随数据更新自动计算
- 当前状态：薄弱（仅 PMARP + RVOL），待补齐

**情绪分析（日频）**
- 驱动方式：经因子研究验证过的情绪因子
- 产出：市场情绪信号（热度、恐惧/贪婪、社交趋势）
- 更新触发：每日随数据更新自动计算
- 当前状态：雏形（Attention Engine），待补齐数据源和分析框架

### 第三层：策略层（信号合成）

汇总三个分析维度，在任意时间点对任意股票产出一个明确的建议仓位与动作。

- 核心框架：OPRMS（将从纯基本面演化为三维合成）
- 核心难题：三个维度"打架"时如何调和（概率思维 + 完整交易系统）
- 输出格式：单票建议仓位 + 动作（建仓/加仓/持有/减仓/清仓）+ 置信度
- 当前状态：OPRMS 存在但仅反映基本面维度

### 第四层：CIO 层（组合管理 + 组合构建）

拆分为两个子能力，建设路径和依赖不同。

**CIO-A：组合管理（对已有持仓的诊断 + 风险评估 + 行为纠偏）**

- 核心问题：**"我现在的组合健康吗？"**
- 核心能力：
  - 组合层风险诊断（敞口、集中度、等效 β、对冲有效性、情景压力测试）
  - 个股层信号聚合（技术预警、成本预警、kill condition 审视）
  - 行为纠偏（损失厌恶、确认偏差、禀赋效应等认知偏差防火墙）
- 输入：**当前真实持仓** + 数据层 + 宏观环境
- 依赖：**仅依赖数据层**（不依赖分析层/策略层的成熟度）
- 定位：只要持有真实仓位就立即需要，不等上游成熟
- 建设状态：Phase 1 已落地 — Portfolio Intelligence 每日推送 + `/trade` skill；下一阶段扩容组合级评估能力（如期权对冲有效性）
- 范围约束：**不涉及新建仓/仓位调整/加减仓建议** — 那属于 CIO-B

**CIO-B：组合构建（从单票建议合成新组合 + 仓位配置）**

- 核心问题：**"我下一步应该买什么、配多少？"**
- 核心能力：N 只股票的独立策略建议 → 一个合理的组合
- 输入：所有策略层建议 + 宏观/Regime 环境 + 当前持仓
- 依赖：**必须等策略层成熟**（需要可靠的单票建议作为输入素材）
- 风格：集中型（5-15 只，单票 15-25%），参照传奇个人投资者
- 建设状态：研究计划已立项（`docs/plans/2026-03-03-cio-layer-research-plan.md`），仍 gated on 分析层 + 策略层成熟度

### 侧翼：宏观/Regime

不属于个股分析，而是定义市场环境，供 CIO 层参考。

- Regime 分类：CRISIS / RISK_OFF / RISK_ON / NEUTRAL
- 跨资产信号：5 个检测器（carry unwind, credit stress, liquidity drain, reflation, risk rally）
- 投资主题/热点：Theme Engine（量价动量 + 注意力量化）

### 离线 R&D：回测引擎 + 因子研究

不在生产管道内，是发现和验证有效因子的实验室。

- 回测引擎：策略级验证（"这个策略赚钱吗？"）
- 因子研究：因子级验证（"这个因子有预测力吗？"）
- 产出：经验证的因子 → 纳入技术分析或情绪分析层

## 各层完成度

| 层 | 完成度 | 关键缺口 |
|---|---|---|
| 数据层 | ~95% | 期权链数据有限（暂缓） |
| 分析层 — 基本面 | ~90% | 基本齐备 |
| 分析层 — 技术 | ~20% | 需更多经验证的指标 |
| 分析层 — 情绪 | ~15% | 数据已有，缺因子定义+分析框架 |
| 策略层 | ~15% | OPRMS 需三维化 |
| CIO 层 — A（组合管理）| ~25% | PI 已上线，待扩容组合级评估（对冲有效性、情景压测） |
| CIO 层 — B（组合构建）| ~5% | 研究阶段，gated on 分析层+策略层 |
| 离线 R&D | ~60% | 需更多因子挖掘 |
| 宏观/Regime | ~70% | 热点/主题引擎待加强 |

## 建设路径

**两条轨道并行推进**，分工清晰：主轨回答"下一步买什么"，副轨回答"现在的组合健康吗"。两者在 Boss 日常工作中同等重要，但主轨必须等基建成熟，副轨可以立即落地。

### 主轨（自下而上，逐层夯实）— 通向 CIO-B

1. **补齐分析层**：技术分析指标扩充 + 情绪因子定义与验证（数据层已就绪）
2. **演化 OPRMS**：从纯基本面评级 → 三维合成策略输出
3. **构建 CIO-B**：从单票建议合成新组合 + 仓位配置

每一步都不跳跃——分析层三条腿站稳了，策略层自然有东西可以合成；策略层有了可靠的单票建议，CIO-B 才有素材做新组合。

### 副轨（已有持仓驱动）— CIO-A 独立建设

只要持有真实仓位就立即可做，**不依赖分析层/策略层的成熟度**。基于数据层 + 基础金融数学（Greeks、Beta、情景分析）即可交付。

1. **PI 数据呈现**（已完成）— 每日推送个股信号 + 组合概览 + kill condition checklist
2. **组合层诊断评估**（当前阶段）— 对冲有效性、情景压力测试、组合层风险归因
3. **行为纠偏工具**（后续）— 认知偏差检测、交易决策复盘

### 路径纪律

- CIO-A 的所有功能**严禁越界触及"新建仓/仓位调整/加减仓建议"** — 那属于 CIO-B，必须等主轨成熟
- CIO-A 只回答"现状如何"，不回答"下一步怎么动"
- 如果某个功能同时横跨诊断和建议两块，拆开：诊断部分走 CIO-A 立即做，建议部分挂在 CIO-B 等主轨
